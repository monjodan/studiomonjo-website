# Studio Monjo Website Optimization Plan

Date: June 9, 2026  
Purpose: translate the refined brand platform into concrete website changes across copy, design, SEO, structure, conversion, and performance.

Core platform:

> From one hand to another.  
> For the art of writing by hand.  
> Writing begins before the words.

## 1. The Website Job

The website should do four things in this order:

1. Make the brand instantly understandable.
2. Make the experience of writing physically desirable.
3. Route visitors to the right action: Naver, Courrier, workshop, Studio story.
4. Give Google, Naver, and social platforms enough clear text and structure to understand the site.

The site should not feel like a normal ecommerce website. It should feel like entering a small writing house.

But it still has to answer:

- What do you make?
- Why does it matter?
- Which product is for me?
- Where do I buy or book?
- Who makes it?

## 2. Current Website Diagnosis

### What already works

- The three-door homepage is distinctive.
- Full-bleed photography gives the site atmosphere.
- The cream, ink, and seal-red palette is close to right.
- The notebooks page already has two useful product families: Robey in Seoul and Monjo Line.
- Courrier has the strongest emotional origin story.
- Workshop page already understands materiality: fold, stitch, paint, sign.
- The site already has canonical URLs, hreflang, OG images, alt text, and some schema.

### What is holding momentum back

1. The homepage phrase "Writing Objects" is too cold and unclear.
2. The homepage does not yet express the full sensory writing experience.
3. The public site and local repo product story may not be aligned live.
4. The site does not have a dedicated Studio/About page for the founder, AI, French writing lineage, and Seoul.
5. The homepage is too thin for SEO if it remains only a full-screen door interface.
6. Product pages sometimes lead with atmosphere before decision clarity.
7. Workshop page has weak conversion because it says bookings open soon without a real waitlist/private-booking action.
8. The existing copy talks about writing, but not enough about pen, ink, paper, thread, object, uniqueness, and gift.

## 3. Priority Order

Do not redesign everything first.

Fix in this order:

1. Homepage copy and door labels
2. Add crawlable homepage sections below the doors
3. Update notebooks page hero and product-family copy
4. Add Studio/About page
5. Update workshop page conversion
6. Tune Courrier copy into the "from one hand to another" platform
7. Improve metadata, schema, sitemap, internal links
8. Performance pass on image loading and JS

## 10. Visual Optimization

### Image criteria

Every important image should answer at least one:

- Can I see the hand?
- Can I see the material?
- Can I feel the act of writing?
- Can I understand what I receive?
- Can I see how this is made?
- Can I see why this is not identical to another object?

### Replace generic mood with material proof

Use more:

- nib touching paper
- ink drying
- thread being pulled
- page opening flat
- handwritten correction
- stamp impression
- envelope crease
- story card insertion
- finished notebook in hand

Use less:

- generic desk styling
- dark images where the object disappears
- empty paper with no invitation
- decorative atmosphere without product proof

### Hero image priority

Create one flagship image:

> dip pen, fountain pen, keyboard, hand-bound notebook, and a hand entering the frame.

Use it on the Studio page and maybe as a lower homepage section, not necessarily the first viewport.

## 13. Performance Optimization

### Likely performance risks

- large full-bleed images
- GIFs on Naver-style pages
- multiple local inline CSS blocks
- Google Fonts
- full-screen image hover stack

### Immediate technical improvements

1. Ensure first hero images use `fetchpriority="high"` only where needed.
2. Keep below-fold images `loading="lazy"`.
3. Prefer WebP for local site images, with JPG fallback.
4. Avoid loading all hover images at full quality if not needed on mobile.
5. Consider generating smaller mobile hero image variants.
6. Add explicit image width/height everywhere to avoid layout shift.
7. Keep JS small and defer noncritical scripts if possible.

### Homepage-specific

The home currently preloads three full-bleed hover backgrounds through markup. Keep the experience, but on mobile consider:

- one default image only
- no hover stack
- simpler door backgrounds

This can improve load and avoid wasting bandwidth on mobile visitors.
